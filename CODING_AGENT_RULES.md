# Coding Agent Rules


Work milestone by milestone.


Do not add features outside the current milestone unless explicitly instructed.


After each milestone:
1. Summarise changes.
2. List files created or modified.
3. Explain how to test from a phone.
4. List automated tests added.
5. List known limitations.
6. Stop and wait for human verification.


Prioritise:
- polished mobile-first UI
- simple maintainable code
- TypeScript strictness
- clear database migrations
- privacy-conscious design
- no mandatory email/password login
- no exposure of API keys in client code
- Vercel preview deployments
- Neon Postgres compatibility


Do not implement:
- books
- albums
- restaurants
- streaming links
- WhatsApp bot
- native mobile app
- public profiles
- payments
- AI recommendations
unless explicitly instructed.


Use:
- Next.js
- TypeScript
- PostgreSQL
- Prisma
- Tailwind
- TMDB API
- Vercel
- Neon
